# 课件自检、修复与发布门禁

课件自检的目标是：先用本地规则发现确定性错误，再用浏览器验证实际显示和交互，
最后只把仍需判断的问题交给质量阶段模型。

## 使用方法

1. 打开已生成或已导入的课堂。
2. 点击课堂顶部的课件检查。
3. 日常选择“均衡”，正式上线前选择“严格”。
4. 点击“完整自检并自动修复”。
5. 查看报告中的场景、问题类型和修复状态。
6. 无法自动修复的问题进入 Pro Mode 修改。
7. 重新运行严格检查。
8. 所有严重问题消失后下载 `.maic.zip`。

## 检查范围

| 类别 | 会检查什么 |
| --- | --- |
| 知识点 | 大纲中的场景是否存在、是否重复、类型是否匹配、关键点是否有明确文字或公式证据 |
| 结构 | Stage、Scene、元素、题目和动作 ID，顺序、类型与引用关系 |
| 文本 | 乱码、非法字符、占位符、“场景 X”“知识点 X”和无意义编号 |
| 公式 | LaTeX 是否为空、是否能被 KaTeX 解析 |
| Quiz | 题型、答案数量、答案映射、重复选项、缺少配图却要求“观察下图”等问题 |
| 讲解词 | 当前场景定位、前后场景依赖、页码引用、平台流程承诺、说话人标签和智能体姓名 |
| 资源 | 本地文件是否存在、路径是否可移植、文件大小、哈希、图片是否能完整解码 |
| Slide | 坏图、字体失败、文本溢出、元素越界、明显重叠、空白或渲染错误 |
| 交互场景 | HTML 加载、脚本异常、控制台错误、外部依赖、空白画面、视口溢出和代表性控件操作 |
| 音频与视频 | 引用是否存在、文件是否可加载、浏览器是否播放失败 |

## 三个档位

| 档位 | 适合 | 截图复核 | 自动修复 | 发布限制 |
| --- | --- | --- | --- | --- |
| `fast` | 旧课件预筛、批量初检 | 关闭 | 关闭 | 报告问题 |
| `balanced` | 日常生成和修改 | 开启并缓存 | 每场景限次 | 严重问题阻断 |
| `strict` | 上线和正式交付 | 开启并缓存 | 更严格的限次修复 | 严重问题和关键警告阻断 |

## 自动修复原则

系统只自动处理边界清楚的问题：

- 补齐缺失或重复的 ID。
- 修正场景顺序、Stage 引用和内容类型。
- 补充交互 HTML 的 doctype。
- 修复可确定的简单 SVG 路径。
- 为场景首段补充当前知识点定位。
- 清理依赖前后场景、具体页码、平台流程或智能体姓名的讲解词。
- 对发生布局、媒体或运行错误的场景执行限次 AI 修复。
- 修复后只复检变化场景。
- 修复结果变差时保留更好的旧版本。

系统不会为了通过检查而擅自改变课程结构、删除知识点、改写 Quiz 答案语义或把交互场景
替换成静态说明。

## 语音可移植性

课件可能被单独打开、重排或用于自适应路线，因此讲解词应满足：

- 首段明确当前知识点或活动目的，不假设前序场景已经完成。
- 场景内部的语音段可以承接本场景已经发生的操作和观察。
- 末段只总结当前场景，不预告固定下一页或下一场景。
- 不引用“第几页”“上一页”“下一个场景”。
- Quiz 不承诺提交后出现教师、助教或讨论流程。
- 不出现“我是某某老师”。
- 不出现“某某同学，你来试试”。
- 不出现“刚才某某助教提到”。
- 不使用“教师：”“AI 助教：”等说话人标签。
- 使用“你”“大家”“学习者”“我们”等中性称呼。

生成时会先通过提示词约束，动作生成后再执行确定性净化。严格模式会检查场景首尾和全部
讲解段；文字发生变化时旧音频引用会被清除，启用 TTS 的完整流程会重新生成对应语音。

## 能保证什么

系统可以可靠发现大部分结构、编码、资源、浏览器渲染和运行时问题，并留下可复核的报告与
截图。存在严重问题时不会生成最终归档。

## 不能自动保证什么

- LaTeX 能显示不代表数学结论一定正确。
- 视觉模型不能替代教材级事实核对。
- 互动检查只操作代表性控件，不能穷举所有输入组合。
- 位图内部的小字和专业结论不一定能被普通文本规则发现。
- 音频默认不做逐句转写，视频默认不做逐帧语义检查。
- 没有原始大纲的旧课件无法完整证明知识点覆盖，只能检查现有内容。

高风险公式、证明、Quiz 答案和教材结论仍应使用原始教材、计算工具或人工复核。

## 报告与证据

证据目录：

```text
data/courseware-audits/<classroomId>/<runId>/
```

主要文件：

| 文件 | 内容 |
| --- | --- |
| `courseware-guard-report.json` | 结构、文本、Quiz、公式和讲解词问题 |
| `courseware-knowledge-report.json` | 大纲与知识点覆盖 |
| `courseware-resource-report.json` | 图片、音频、视频和本地资源 |
| `courseware-visual-report.json` | Slide 浏览器与截图复核 |
| `courseware-interactive-report.json` | 交互场景运行结果 |
| `screenshots/` | Slide 检查截图 |
| `interactive-screenshots/` | 交互场景检查截图 |

## 成本控制

- 结构、文本、公式语法、Quiz、知识合同和资源检查不调用模型。
- 互动运行检查使用本地浏览器。
- 截图复核按内容哈希缓存。
- 未变化场景不重复检查。
- 相同问题不重复调用修复模型。
- 每个场景和每轮任务都有修复次数上限。
- 日常先用 `balanced`，最终发布再用 `strict`。

## 推荐配置

```dotenv
OPENMAIC_COURSEWARE_AUDIT_PROFILE=balanced
OPENMAIC_COURSEWARE_MAX_REPAIR_PASSES=2
OPENMAIC_COURSEWARE_MAX_REPAIRS_PER_SCENE=1
OPENMAIC_COURSEWARE_VISION_MAX_OUTPUT_TOKENS=1200
OPENMAIC_COURSEWARE_VISION_CACHE=true

OPENMAIC_COURSEWARE_INTERACTIVE_AUDIT=true
OPENMAIC_COURSEWARE_INTERACTIVE_EXERCISE=true
OPENMAIC_COURSEWARE_INTERACTIVE_CONCURRENCY=3
```
